Introduction

EnSaaS Web App Scanner is a security tool that can perform advanced attacks against web applications in order to find security vulnerabilities. The scanner supports both passive scan and active scan. Passive scan spiders the target's directories and files in order to find insecure issues and active scan performs more sophisticated attacks by sending malicious strings to the target in order to find more vulnerabilities. It also integrates with WISE-PaaS/Dashboard for real-time data visualization. User can view their scanning history on scanning portal at any time in order to keep track of the security issues regarding their owned web applications.

Initial Setup

Once EnSaaS Web App Scanner is deployed in your space, there are some configurations need to be set at the very first time.

Database Binding (Required)

First, you need to bind a mongo-db service to our Web-App-Scanner. This could be done by go to the "Service Instance List" → Click "mongodb" → "Credentials" → Create a key → "Application Bind" → Click our app →Enter your key.

Dashboard URL Setup (Required)

The URL could be found on your WISE-PaaS Management Portal. Please make sure you've already subscribed and had WISE-PaaS/Dashboard in your space. You must have admin access (not a viewer) to your Dashboard before setup, otherwise, the configuration would fail.

Then, go to the Web App Scanner to setup your dashboard.

Notification Setup (Optional)

Our service integrates with WISE-PaaS/Notification, a summary report would automatically send to your mailbox whenever a security scan is done. If you want to have this feature, you should configure your email service setting also.

Scanning Portal

Intro

You can manage your security scans in the scanning portal. Enter your target's URL in the field and click "Scan" to start scanning. If others in your organization are using the scanner, the scanning portal would inform you and put your request in the scanning queue.

Passive Scan

EnSaaS Web App Scanner performs a passive web security scan by spidering the targeted website, looking for directories, files, and HTTP methods, and then detecting issues like insecure HTTP headers, insecure cookie settings, and a few others.

  • Advanced Options
    • SubtreeOnly : Enable to restrict the spider under the target url subtree.
    • Recurse : Enable to use the nodes underneath the one specified target to seed the spider.

Full Scan

When a full scan is configured. EnSaaS Web App Scanner would perform an active scan right after finishing a passive scan, and it includes detection of SQL Injection, XSS, Local File Inclusion, Command Injection, for a more comprehensive security assessment.

  • Advanced Options
    • Alert Threshold : Set the Alert Threshold (Default is Medium)
    • Attack Stregth : Set the Attack Stregth (Default is Medium)
    • inScopeOnly : Set In Scope only to true to constrain the scan to URLs that are in scope.
    • Recurse : Set recurse option to scan URLs under the given target URL.

Scheduled Scan

You can schedule your scan against certain website at a specific time, and also choose whether to repeat the scan periodically. If others in your organization is currently using the scanner, your security scan would also be put in the scanning queue.

Scan History

You can view your scan history below the scanning portal, and click the link of a dashboard to review your scanning summary. And if your scanning requests are still in the process, you can also check the status of your requests here.

Dashboard

Intro

WISE-PaaS/Dashboard is a service that gives you a clean interface to keep track of all the information in one place. We integrate EnSaaS Web App Scanner with Dashboard to give you the best security assessment with real-time visualization.

Panels

Scanning progress panel shows the current progress of your scan. If you chose "Passive Only" in your scan option, the "Active Scan" progress would consistently be 0 by default.

Pie chart and table panel shows the count of High, Medium, Low and Informational issue that has already discovered in your system. They keep dynamically updating until your scan is finished.

You can download the scanning report once the scan is finished. The same file can also be found in the "Scan History" on your Scanning Portal.

Summary Report panel dynamically shows the newest security issues found with explanations in details. The context presented would be the same as the report provided in the download button. However, we highly suggest you download the scanning report in order to do further security maintaining to your system.

Report

Intro

The first table of the report shows the summary of alerts, followed by alert details below. The alert details show descriptions of certain vulnerabilities with URL, Method, and Parameter. Moreover, the suggested solution and security references are also provided.
